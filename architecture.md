# Архитектура проекта Sunberg (FSD)

Этот документ определяет архитектурный стандарт, используемый для разработки платформы Sunberg. Мы следуем методологии **Feature-Sliced Design (FSD)** для обеспечения модульности, тестируемости и легкости поддержки.

## 1. Слои Архитектуры (Layers)

Проект разделен на иерархические слои, где каждый слой может импортировать данные только из слоев, находящихся ниже:

1.  **app**: Инициализация приложения. Настройки роутинга (`App.jsx`), глобальные провайдеры и стили (`src/app/styles/`).
2.  **pages**: Композиционные компоненты. Страницы собираются исключительно из виджетов. Логика на этом уровне минимальна.
3.  **widgets**: Автономные бизнес-блоки. Самый важный слой в текущей миграции. Примеры: `Header`, `DeepDive`, `Hero`.
4.  **features**: Интерактивные части с бизнес-ценностью. Примеры: `ContactForm`, `SolarCalculator`.
5.  **entities**: Бизнес-сущности. Минимальные блоки данных и UI. Примеры: `ProjectCard`, `ServiceItem`.
6.  **shared**: Переиспользуемый фундамент. UI-Kit (кнопки, инпуты), иконки, вспомогательные функции (libs).

## 2. Принцип Public API

Каждый модуль (особенно в `widgets/`) должен иметь файл `index.js`, который является его "входной дверью".
- Извне можно импортировать **только** то, что экспортировано в `index.js`.
- Запрещено обращаться к внутренностям модуля (например, напрямую к `ui/Component.jsx`).

## 3. Стилизация: CSS Modules + BEM

Мы используем **CSS Modules** для полной изоляции стилей компонентов.
- Файл стилей именуется как `ComponentName.module.css`.
- Внутри модуля можно использовать простые имена классов или BEM-нотацию.
- Использование глобальных селекторов внутри модулей допускается только через `:global(.global-class)`.

## 4. Глобальные Стили (Design Tokens)

Все константы дизайна хранятся в `src/app/styles/variables.css`:
- Цвета (`--color-primary`, `--color-bg` и т.д.)
- Сетки и отступы.
- Шрифты и радиусы скругления.

## 5. Правила миграции

При переносе компонента из `src/components/` (Legacy) в `src/widgets/` (FSD):
1.  **Не удалять** сразу старые стили в `index.css` — только комментировать.
2.  Создавать изолированный `.module.css`.
3.  Использовать `index.js` для экспорта.
4.  Обновлять пути импорта в `src/pages/`.
